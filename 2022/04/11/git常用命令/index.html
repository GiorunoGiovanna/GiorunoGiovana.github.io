  10. 暂存更改

    git stash可以将修改暂时存放起来

    应用场景：当当前分支已经进行更改，但是没有准备提交，而需要临时切换到其他分支，由于修改没有做处理，是不允许切换分支的，这个时候就可以使用git stash将代码暂时存放起来，以便切换分支，等重新回到当前分支的时候再使用。

  11. 撤销修改

    git reset(可以回退commit操作)

    • 撤销整体的修改

      参数soft指的是：保留当前工作区，以便重新提交
      还可以选择参数hard，会撤销相应工作区的修改，一定要谨慎使用

    • 撤销某次修改

    撤销已经push到远程仓库的提交

    必须添加参数force进行强制提交，否则会提交失败，并报错。报错原因：本地项目版本号低于远端仓库版本号。

  12. git merge和git rebase的区别

    git merge和git rebase都是用于合并分支使用的。

    • git merge

      git merge是将两个不同的分支在merge的主分支上新开一个提交，在这个提交中会合并两个分支，并且留下记录，对于团队合作来说更好。

    • git rebase

      rebase 会将整个 feature 分支移动到 master 分支的顶端，从而有效地整合了所有 master 分支上的提交。不会留下分叉，比较美观

  13. git cherry pick

    前面的git merge和git rebase都是针对于整个分支的合并过程，但是有时候我们不需要将一个分支的所有提交都合并，只想合并其中某些分支，这个时候就可以使用git cherry pick

    如果在合并过程中发生冲突，有以下方法

    • 解决冲突，然后git add，然后继续合并git cherry-pick --continue
    • 回滚：git cherry-pick --abbort
    • 中断：git cherry-pick --quit
