面经摘抄03

  1. 怎么知道点击的横纵坐标

    首先给点击绑定事件监听，然后根据回调中的事件对象中的offsetX和offsetY判断点击发生位置

  2. vue依赖收集在什么时候发生，父子组件都会更新吗

    vue依赖收集是为了在接受到数据更改的时候能够通知到其他使用了这个数据的组件的一种手段，通常在初始化阶段就发生依赖收集，在Vue2中，由于响应式原理的限制，vue2对于一些能增改数组的方法没有很好的监听手段，于是需要重写数组方法，在这个过程中也会发生依赖收集。

    父子组件都会进行依赖收集和更新，但是更新的顺序和具体实现有关，不一定是所有的父组件都会在所有的子组件更新之前更新。通常情况下，父组件会在子组件之前更新，但也可以通过一些特殊的方式来改变更新的顺序。

  3. for循环1000次，组件会更新多少次

    会更新1次，因为Vue更新视图不是同步的而是异步更新，这个是根据当前浏览器是否支持promise或setTimeout来决定的。由于for是同步的，那么会等到同步代码全部结束后再根据最后的结果触发watcher走响应式更新的一套操作。

  4. 框架是怎么知道什么时候生成虚拟dom
  5. 如何终止forEach遍历？

    1. break(X)

    2. return false(X)

    3. 抛出异常

  6. for…in和for…of的区别

    for…in是用于遍历可枚举属性，包括它原型链上新增的可枚举属性

    for…of本质上是遍历拥有迭代器iterator的对象，而普通对象没有迭代器，因此可以给普通对象添加一个迭代器来使其可以被遍历

  7. sessionStorage真的不能在页面之间共享状态吗？

    标准答案是不能，但是在标签或窗口中打开一个同源新页面时会复制，注意是复制一份全新的副本作为新会话窗口的上下文，也就是说在打开之前的状态是保持一致的，但是打开之后就不能共享实时的状态了

  8. 跨页面通信方法

    1. 使用localStorage进行同源页面跨页面通信

    2. 使用BroadCast Channel进行同源页面跨页面通信

      可以创建一个用于广播的通信频道。类似于localStroage。

    3. 使用Service Worker进行同源页面跨页面通信

    4. 使用iframe桥进行非同源页面跨页面通信

      要实现该功能，可以使用一个用户不可见的 iframe 作为“桥”。由于 iframe 与父页面间可以通过指定origin来忽略同源限制，因此可以在每个页面中嵌入一个 iframe （例如：http://sample.com/bridge.html），而这些 iframe 由于使用的是一个 url，因此属于同源页面，其通信方式可以复用上面第一部分提到的各种方式。

    5. 使用webSocket进行跨页面通信

  9. 不使用额外变量交换a，b值
  10. JavaScript死锁和死锁检查

    一般情况下，认为死锁是两个或以上的进程发生冲突导致不断等待的情况产生的。但是JavaScript作为经典的单线程语言也可以发生死锁。利用Promise没有决议就会一直阻塞的原理，可以实现阻塞部分代码形成死锁

  11. 检查内存泄漏

    使用chrome浏览器开发者工具中的memory或者performance检查浏览器变量情况。

  12. 如何实现组件按需引入
    • TreeShaking（antD）基于webpack打包过程的treeShaking操作除掉多余的无用代码达到按需引入的目的，好处是不需要额外配置，坏处是由于基于webpacck，无法对css做按需引入处理。
    • Babel-plugin-import（elementui、vant）借助Babel解析编辑ATS的能力实现了一种成熟的组件按需引入方式，目前很多的组件库都是依靠这种方式进行了实现，例如Element-UI等。这种方式的优点是不需要库使用者了解原理，只需按照文档说明配置即可。缺点是使用者仍旧需要为了按需引入来增加可能本来无需使用的Babel及其相关配置（这点也是目前仍然有大量程序员不使用组件按需引入功能的原因，不是所有人都想触碰Babel配置的，尤其当他还不够了解Babel机制的时候）。
    • 最后，我们进入到实现按需引入的终极方式——Monorepo！为什么说它是终极方式呢？因为通过Monorepo形式组织的组件库在发布时每个组件都是一个单独的NPM包，用户可以自己选择希望使用的组件并安装（你没听错，甚至都不用安装整个组件库），这样自然做到了完完全全的按需引入（甚至可以说从根本上干掉了这个概念）！

发表于 2023-03-19 更新于 2023-03-20 分类于 Webpack Valine：

webpack热更新原理

在我们传统webpack项目中，如果发生文件上的修改，那么webpack会做出这样的反应，首先会重新编译生成新的bundle文件，然后通知浏览器进行reload刷新重新请求bundle，达到更新web项目的目的。这样的方式在大型项目中会非常吃力，因为每次都要重新请求文件，并且导致当前状态丢失的问题。因此出现了webpack热更新的解决方案。

一句话概括就是热更新允许浏览器在不刷新的前提下，通过替换某些需要被更新的模块来达到热更新的目的，从而不必让当前状态丢失。

基本原理

  • 在webpack中，每次对项目进行编译都会生成一个唯一的hash值，这个hash值可以用来判断文件是否发生了更新。
  • 基于这一点，webpack有一个webpack-dev-server服务器功能，可以开启一个链接webpack和浏览器的websocket链接，用来通知浏览器是否发生了修改。
  • 如果发生了修改，则发起带有新hash的请求，获得更新列表。
  • 再通过jsonp的方式获取到最新的模块代码
  • webpack的HotModulePlugin会对比新旧模块，来决定是否更新

小结

Webpack 的 webpack-dev-server 是 Webpack 生态链上很重要的一环，在我们日常的开发环境，我们可以使用 webpack-dev-server 启动本地服务器，而且能够实现 API 接口代理、静态资源服务器、HMR，甚至还能够通过编写 Express 中间件的方式来扩展功能。

但是 webpack-dev-server 本身也有它的局限性，比如我们项目本身就有一个 Node.js 的业务服务，那么在使用 webpack-dev-server 来模拟接口数据就显得多此一举了。

ESM和CommonJS

在JavaScript的发展历史中，出现了多种模块化规范，这是由于js项目过于庞大的话可以进行解耦合，从而让JavaScript项目更加规范化的目的。其中最为常见的就是node本身实现的CommonJS和es6标准的ES Modules两种规范。

总结主要区别有：

  1. commonjs输出的是值的拷贝和esm输出的是值的引用
  2. commonjs是运行时加载属于同步加载，esm是编译时加载属于异步加载
  3. esm 存在 export/import 提升

CommonJS

CommonJS是最常用的js模块化规范之一，也是nodejs支持的模块规范。其中最重要的概念是module对象和require函数。

module

module是一个对象，专门用于存储js输出的值。其中有三个属性

  • exports：就是module.exports对应的值
  • loaded：表示当前模块是否加载完成
  • paths：node模块的加载路径

Vue3源码学习笔记—生命周期

跟Vue2的生命周期相比，Vue3的生命周期没有过多的变化，无非就是把beforeDestroy改名为beforeUnmounted这种命名统一化的操作，还有新增了几个服务端渲染的生命周期。

为了适应composition API，生命周期也从原来的挂载在Vue实例中变为了可拆卸的模块。

injectHook

用于把对应模块中的我们写的逻辑储存起来，在组件渲染的各个阶段去执行，先暂停组件的任务，把这个钩子执行完成再向下执行。

onBeforeMount 和 onMounted

组件的挂载一共有两种情况，第一种是组件初始化第一次挂载，这个使用就要执行onBeforeMount 和 onMounted和一些初始化的逻辑，还有一种就是组件更新时的重新挂载，这个时候就不必进行其他初始化操作了，就走另一条逻辑进行patch。

Vue3源码-watch、watchEffect、computed

Vue3提供了三种不同的可以监听数据的方法，分别是watch、watchEffect、computed。他们之间的取别在于，

  • watch接收一个参数作为监听对象，同时能够执行回调，并且是默认懒启动的。
  • watchEffect直接接收回调函数，回调函数中使用的对象都会被监听，并且是立即执行的
  • computed和watchEffect有一点像，直接监听回调函数中使用的对象，但是不同的是它返回一个值，这个值是回调函数计算出来的。

由于Vue3支持了composition API，为了组件化所有功能，因此将watch、watchEffect拆分到一个模块中。

可以看到，watch和watchEffect其实是公用一个dowatch逻辑的，因为两个方法除了监听对象和启动时机之外的底层逻辑是相同的。
